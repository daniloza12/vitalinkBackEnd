  ## Contexto
  El backend ya tiene listos estos endpoints:
  - GET  /api/v1/auth/activate?token=xxx        вҶ’ activa la cuenta, responde {
  message: string }
  - POST /api/v1/auth/resend-activation?email=xxx вҶ’ reenvГӯa el email, responde {
   message: string }

  Cuando un usuario se registra, recibe un email con un botГіn que lo redirige a:
  http://localhost:4201/activate?token=<token>

  El status de la cuenta cambia de REGISTRADO вҶ’ ACTIVO al activar.
  Solo cuentas con status REGISTRADO o ACTIVO pueden hacer login (el backend
  devuelve 401 si no).

  ## Lo que necesitas implementar

  ### 1. Ruta /activate
  - Componente ActivateAccountComponent
  - Al inicializar lee el query param `token` de la URL
  - Llama al backend GET /api/v1/auth/activate?token=xxx
  - Muestra estado de carga mientras espera
  - вң… Гүxito: redirige a /login con queryParam activated=true
  - вқҢ Error: muestra mensaje de error con opciГіn de reenviar email (pide el
  email al usuario y llama a POST /api/v1/auth/resend-activation?email=xxx)

  ### 2. Ruta /login вҖ” ajustes
  - Si llega queryParam activated=true, mostrar un banner/alert de Г©xito:
    "ВЎCuenta activada correctamente! Ya puedes iniciar sesiГіn."
  - Si el backend devuelve 401 con message "Account is not active", mostrar:
    "Tu cuenta no estГЎ activa. Revisa tu correo o solicita un nuevo enlace."
    + botГіn "Reenviar email de activaciГіn" que muestre un input de email y llame
   a POST /api/v1/auth/resend-activation?email=xxx

  ### 3. AuthService вҖ” mГ©todos nuevos
  Agregar en el AuthService existente:
  - activateAccount(token: string): Observable<{ message: string }>
  - resendActivationEmail(email: string): Observable<{ message: string }>

  ### 4. Ruta /register вҖ” ajuste post-registro
  DespuГ©s de un registro exitoso, en lugar de ir directo al login, mostrar
  pantalla/mensaje:
  "Te enviamos un email a <correo>. Revisa tu bandeja y haz clic en el enlace
  para activar tu cuenta."
  Con opciГіn de reenviar si no llegГі.
